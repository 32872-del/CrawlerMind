# 2026-05-06 10:20 - Result CLI

## Goal

Own the Storage/CLI module and add a shared command-line entrypoint for reading
persisted crawl results.

## Changes

- Added `run_results.py`.
- Supported commands:
  - `list`
  - `show <task_id>`
  - `items <task_id>`
  - `export-json <task_id> output.json`
  - `export-csv <task_id> output.csv`
- Added `--db-path` for tests and portable environments.
- Configured CLI stdout/stderr as UTF-8 to avoid PowerShell Chinese mojibake.
- Added CLI tests in `autonomous_crawler/tests/test_run_results_cli.py`.
- Updated `README.md`, `PROJECT_STATUS.md`, and short-term plan.

```text
python run_results.py items 6a164795 --limit 3
python run_results.py export-json 6a164795 <temp-json-path>
```

## Result

Persisted workflow output is now inspectable without opening SQLite or calling
the API directly. This gives future Codex sessions a simple shared observation
tool.
