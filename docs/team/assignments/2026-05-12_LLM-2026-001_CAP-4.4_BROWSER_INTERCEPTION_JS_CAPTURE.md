## Goal

Implement the first Browser Interception MVP so CLM can inspect and control
page resources instead of only rendering pages passively.

This is an advanced crawler capability task. Do not make it a site-specific
scraper.

## Why This Matters

Enterprise crawler developers need to know:

- which JS bundles loaded
- which XHR/fetch/WebSocket calls happened
- which resources can be blocked to speed crawling
- where hooks or init scripts can be injected later

This is the entry point for later JS hook, anti-debug analysis, and signature
parameter tracing.

## Requirements

Create a module that supports:

1. A config object for interception:
   - block resource types: image, media, font, stylesheet
   - capture JS responses
   - capture API-like responses
   - optional init script string
2. A result object with:
   - final URL
   - resource counts by type
   - blocked URL list
   - captured JS asset list: URL, status, content type, size, sha256
   - captured API list: URL, method, status, content type
   - errors
3. Playwright implementation using `page.route()` and response listeners.
4. Deterministic tests with mocked Playwright. No external network required.
5. Sensitive headers or cookies must not appear in output.
