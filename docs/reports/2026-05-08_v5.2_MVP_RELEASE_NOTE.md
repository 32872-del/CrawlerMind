# Crawler-Mind v5.2 MVP Release Note

## Summary

v5.2 is a real MVP release, not a skeleton. It can run end to end on supported
targets, persist results locally, and optionally use LLM advisors for Planner
and Strategy decisions. Deterministic fallback remains the default.

## What It Can Do

- Run the core workflow:
  `Planner -> Recon -> Strategy -> Executor -> Extractor -> Validator`
- Crawl deterministic fixtures and supported public pages.
- Use HTTP or browser fallback when appropriate.
- Collect and validate Baidu realtime hot-search results.
- Persist task results to local SQLite.
- Expose a local FastAPI service for crawl requests and history lookup.
- Run optional OpenAI-compatible LLM advisors through CLI, config, or
  request-level FastAPI input.
- Check LLM configuration before crawling with:
  `python run_simple.py --check-llm`

## What It Cannot Yet Do Reliably

- It does not guarantee success on arbitrary websites.
- It does not fully solve dynamic-page coverage beyond the tested browser
  fallback cases.
- It does not implement API interception end to end.
- It does not provide Cloudflare bypass or challenge circumvention.
- Its FastAPI jobs are still in-memory and are lost on process restart.
- It does not yet have durable site memory or self-healing crawl loops.

## Known Limitations

- Dynamic pages: browser fallback exists, but broader real-world coverage is
  still limited.
- Cloudflare: detection and diagnosis are supported, bypass is not.
- API interception: the graph does not yet fully integrate this path.
- In-memory jobs: FastAPI background jobs are not durable across restarts.

## How To Start

1. Install dependencies:

```powershell
python -m pip install -r requirements.txt
```

2. Copy and edit the LLM config if you want advisor mode:

```powershell
Copy-Item clm_config.example.json clm_config.json
notepad clm_config.json
```

3. Check the LLM config before a crawl:

```powershell
python run_simple.py --check-llm
```

4. Run a normal crawl:

```powershell
python run_simple.py "collect product titles and prices" https://example.com
```

5. If you have no API key yet, use the mock fixture:

```powershell
python run_simple.py "collect product titles and prices" mock://catalog
```

## LLM Configuration

Supported CLI/config keys:

- `CLM_LLM_BASE_URL`
- `CLM_LLM_MODEL`
- `CLM_LLM_API_KEY`

Or via `clm_config.json`:

```json
{
  "llm": {
    "enabled": true,
    "base_url": "https://api.openai.com/v1",
    "model": "gpt-4o-mini",
    "api_key": "replace-with-your-api-key",
    "use_response_format": true
  }
}
```

`python run_simple.py --check-llm` validates the config, resolves the endpoint,
and reports provider issues without exposing the key.

## Best Current Use Cases

- Stable fixture crawls
- Baidu realtime hot-search smoke
- Early service integration tests
- Local LLM advisor experiments with deterministic fallback

## Notable Portability Rule

Runtime artifacts are intentionally excluded from portable project state:

- `autonomous_crawler/storage/runtime/`
- `autonomous_crawler/engines/runtime/`
- `dev_logs/skeleton_run_result.json`

## Status

This release is a strong local MVP with real smoke coverage and optional LLM
advisor support. It is not yet a full autonomous crawler platform.
